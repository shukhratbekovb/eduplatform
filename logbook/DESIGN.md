### 1.3 Типографика

Идентична CRM: Inter, те же размеры (`text-xs` → `text-3xl`), те же веса.

**LMS-специфичные применения:**
```
Lesson title:      text-sm / font-semibold / gray-900
Lesson time:       text-xs / font-medium / gray-500
Student name:      text-sm / font-semibold / gray-900
Risk label:        text-xs / font-medium (цвет по токену риска)
Diamond count:     text-sm / font-bold / diamond (#6366F1)
Coin balance:      text-2xl / font-bold / coin (#F59E0B)
Grade value:       text-lg / font-bold / gray-900
Grade low (<6):    text-lg / font-bold / danger-500
```

### 1.7 Анимации

Те же что в CRM + LMS-специфичные:

```typescript
// Унаследованные из CRM
animate-fade-in:        opacity 0→1, 200ms ease-out
animate-scale-in:       scale 0.95→1 + opacity 0→1, 150ms ease-out
animate-slide-in-right: translateX 20px→0 + opacity 0→1, 250ms ease-out

// LMS-новые
animate-diamond-pop: {
  // При получении бриллианта — pop-анимация иконки
  '0%':   { transform: 'scale(1)' },
  '50%':  { transform: 'scale(1.4)' },
  '100%': { transform: 'scale(1)' },
  duration: 350ms, easing: spring
}

animate-risk-critical-pulse: {
  // Пульсация красной точки для Critical риска
  '0%, 100%': { opacity: 1, transform: 'scale(1)' },
  '50%':      { opacity: 0.5, transform: 'scale(1.3)' },
  duration: 2s, iteration: infinite
}

animate-lesson-inprogress: {
  // Мягкое свечение карточки урока "в процессе"
  '0%, 100%': { boxShadow: '0 0 0 0 rgba(245,158,11,0.4)' },
  '50%':      { boxShadow: '0 0 0 6px rgba(245,158,11,0)' },
  duration: 2s, iteration: infinite
}

animate-pool-deplete: {
  // Когда бриллиант расходуется из пула
  transition: background-color 200ms ease, transform 150ms ease
}
```

## 2. Information Architecture

```
LMS Portal
├── /login                              — Вход в систему
│
└── /                                   — LMS Shell (sidebar + topbar)
    ├── /schedule                       — Расписание (Kanban Пн–Вс)
    │
    ├── /groups                         — Список групп
    │   └── /groups/:id                 — Детальная страница группы
    │
    ├── /students                       — Список студентов
    │   └── /students/:id               — Профиль студента (7 вкладок)
    │
    ├── /lessons/:id                    — Проведение урока (Teacher/MUP)
    │
    ├── /homework                       — Домашние задания (очередь учителя)
    │
    ├── /late-requests                  — Поздние запросы
    │   (Teacher: свои / MUP: входящие на рассмотрение)
    │
    ├── /tasks                          — Задачи МУП (Kanban)
    │   (только MUP / Director)
    │
    ├── /analytics                      — Аналитика LMS
    │   (только MUP / Director)
    │
    ├── /compensation                   — Компенсации учителей
    │   (только Director / MUP)
    │
    └── /settings                       — Настройки
        (только Director / MUP)
        Tabs: Направления | Предметы | Группы | Кабинеты
```

### Секции и назначения

| Секция | Назначение | Director | MUP | Teacher |
|---|---|---|---|---|
| Расписание | Канбан расписания по дням | ✅ все | ✅ все | ✅ свои |
| Группы | Управление группами и студентами | ✅ | ✅ | ✅ (свои) |
| Студенты | Профили, риск, зачисления | ✅ | ✅ | ✅ (свои группы) |
| Уроки | Проведение: посещаемость + оценки | ✅ | ✅ | ✅ (свои) |
| Домашние задания | Очередь непроверенных работ | ✅ | ✅ | ✅ |
| Поздние запросы | Teacher подаёт / MUP рассматривает | ✅ | ✅ | ✅ |
| Задачи МУП | Академические задачи | ✅ | ✅ | ❌ |
| Аналитика | Посещаемость, оценки, риски | ✅ | ✅ | ❌ |
| Компенсации | Модели оплаты учителей | ✅ | ✅ | ❌ |
| Настройки | Направления, предметы, кабинеты | ✅ | ✅ | ❌ |

## 3. Navigation Model

**Тип:** Hybrid (Sidebar + Topbar) — идентичная структура CRM

### Sidebar

```
[🎓 EduPlatform] [LMS]
──────────────────────────
[📅] Расписание
[👥] Группы
[🧑‍🎓] Студенты
[📚] Домашние задания
[⏰] Поздние запросы     ← badge: N (только MUP: количество pending)
──────────────────────────
[✅] Задачи              ← только MUP / Director
[📊] Аналитика           ← только MUP / Director
[💰] Компенсации         ← только MUP / Director
──────────────────────────
[⚙️] Настройки           ← только MUP / Director
──────────────────────────
[← Свернуть]
```

**Sidebar токены:** идентичны CRM (`sidebar-bg: #1E1B4B`, `sidebar-active: #312E81` и т.д.)

**Pending badge на "Поздние запросы":**
- Красный круглый badge `danger-500` с числом
- Появляется только у MUP / Director при наличии pending запросов
- Обновляется в реальном времени (polling или WS)

### Topbar

```
[Название страницы / Breadcrumb]        [🔔 N] [Аватар + Имя ▼]
```

- Высота: 64px, фон белый, нижняя граница `gray-200`
- **Уведомления:** dropdown с последними 5, кнопка "Все уведомления"
- **User menu:** Профиль / Сменить роль (если Director) / Выйти
- **Breadcrumb:** для вложенных страниц (Groups → Группа "React-01")

### Переключение порталов
Director видит выпадающий selector рядом с логотипом: CRM / LMS / Finance

- Teacher: только свои уроки

## 5. User Flows

---

### Flow 1 — Teacher: Провести урок

```
[/schedule] — Teacher видит свои уроки за неделю
  → Видит карточку: React-01, 2 апр, 09:00 [● Идёт сейчас]
  → Click на карточку → /lessons/abc123

[/lessons/abc123]
  → Видит banner: "Окно ввода открыто до 23:59 сегодня ✅"
  → Step 1: Заполнить тему урока (textarea)
  → Step 2: AttendanceTable
    → Для каждого студента выбирает: Вовремя / Опоздал / Отсутствует
    → [Опционально] оставить заметку
  → Step 3: Оценки (опционально)
    → Вводит оценки 1–10
    → Если < 6 → появляется обязательный textarea для комментария
  → Step 4: Бриллианты (опционально)
    → DiamondPool показывает: ◆◆◆◆◆ (5 доступно)
    → Нажимает + рядом со студентом → бриллиант списывается из пула
    → Пул обновляется в реальном времени: ◆◆◆◆○ → ◆◆◆○○
  → Step 5 (опционально): Домашнее задание
  → Step 6 (опционально): Материалы
  → Click "✅ Провести урок"
  → Confirm dialog: "Данные урока будут зафиксированы"
  → Success toast + redirect: /schedule
  → Карточка урока: [✅ Проведён]
```

---

### Flow 2 — Teacher: Запрос на позднее внесение

```
[/lessons/:id] — урок прошёл вчера, status = incomplete
  → Banner: "⚠️ Окно закрыто. Подать запрос на позднее внесение →"
  → Click → открывается LateRequestForm modal

[LateRequestForm]
  → Textarea: причина запроса (обязательно)
  → Click "Подать запрос"
  → Toast: "Запрос отправлен МУП"
  → Banner на уроке: "⏳ Запрос на рассмотрении у МУП"

[Ожидание — МУП одобряет]

[Уведомление Teacher]
  → Bell icon: "МУП одобрил запрос на ввод данных"
  → /lessons/:id → Banner: "✅ Доступ открыт ещё 23 ч 45 мин"
  → Форма ввода разблокирована → Teacher заполняет данные → Провести урок
```

### Flow 3 — MUP: Создать расписание (Single)

```
[/schedule]
  → Click "+ Создать урок" в header
  → LessonForm modal открывается

[LessonForm]
  → Выбирает Направление → фильтруется список Предметов
  → Выбирает Предмет → фильтруется список Групп
  → Выбирает Группу
  → Выбирает Учителя (из активных учителей этого предмета)
  → Выбирает Кабинет
  → Устанавливает дату + время начала/конца
  → Click "Создать урок"

  [Сценарий A — нет конфликтов]
    → Урок создан, карточка появляется в нужной колонке
    → Toast: "Урок создан"

  [Сценарий B — конфликт]
    → ConflictAlert появляется в форме:
      ⚠️ "Учитель Алишер занят: 09:00–10:30, HTML-01"
      ⚠️ "Студент Анвар: занят в группе React-02 в то же время"
    → Форма остаётся открытой
    → MUP изменяет время или кабинет → повторная попытка
```

## 8. UX Logic Considerations

---

### 8.1 Окно редактирования урока

**Правило:** только сегодняшний урок до 23:59

**UI реализация:**
```
Состояние урока             → Что видит Teacher
─────────────────────────────────────────────────
Будущий урок                → Страница read-only, "Урок ещё не начался"
Сегодня, до 23:59           → ✅ Banner "Окно открыто до 23:59", форма активна
Сегодня, 23:59–00:00        → Урок автоматически → incomplete, window closes
Вчера или раньше, incomplete→ ⚠️ Banner + кнопка "Подать запрос"
Вчера или раньше, conducted → ✅ Read-only просмотр данных
Approved late request       → ✅ Banner с countdown + форма активна
```

**Важно:** Frontend проверяет `lesson.date === today` — это UX guard.
Backend также проверяет — frontend не является единственной защитой.

---

### 8.2 Конфликт расписания

**Когда показывать:** при попытке создать/редактировать урок → backend возвращает 422

**ConflictAlert дизайн:**
```
┌─────────────────────────────────────────────────────┐
│ ⚠️  Найдены конфликты расписания                    │
│ ─────────────────────────────────────────────────── │
│ 👤 Учитель: Алишер занят 09:00–10:30 в HTML-01      │
│ 🚪 Кабинет 101: забронирован 09:00–11:00            │
│ 🧑‍🎓 Студент Анвар: занят в React-02 (то же время)  │
└─────────────────────────────────────────────────────┘
```

Поведение:
- Alert появляется **внутри формы** (не toast) — пользователь видит и форму и конфликт
- Кнопка "Создать" становится disabled
- Пользователь должен изменить время/учителя/кабинет — alert исчезает после изменения
- Для bulk-расписания: показывает все конфликтные даты + опция "Пропустить конфликты"

---

### 8.3 Оценка < 6 — обязательный комментарий

**Поведение:**
- Пользователь вводит оценку → если значение < 6 → `animate-slide-in` показывает textarea
- Textarea: placeholder "Опишите проблему или дайте рекомендацию…"
- Кнопка "Провести урок" disabled пока comment не заполнен для всех grades < 6
- Красная рамка на GradeInput + красный label "Комментарий обязателен"
- После заполнения — рамка становится green (валидна)

**Tooltip при hover на disabled кнопке:**
"Заполните комментарии для оценок ниже 6"

---

### 8.4 Риск-система — read-only UI

**Правила:**
- Никогда не показывать кнопку "Пересчитать риск"
- Всегда показывать `riskLastUpdated` — "Обновлено сегодня в 02:00"
- Если `riskLastUpdated` > 26 часов назад → показывать предупреждение:
  `⚠️ "Данные могут быть устаревшими — расчёт ещё не выполнен"`
- Риск-факторы показывают конкретные числа (attendancePercent14d, avgGradeLast5 и т.д.)

**Анимация при изменении риска:**
- При переходе Normal → At Risk: `animate-risk-change` (background flash warning)
- При переходе → Critical: `animate-risk-change` (background flash danger) + badge начинает пульсировать

---

### 8.5 Diamond Pool — реальное время

**Правила:**
- Пул обновляется мгновенно на frontend (local state, не waiting server)
- Сохранение на backend — при финальном submit всего урока
- Если sum(diamonds) > 5 → форма невалидна (Zod validation)
- Если какой-то студент имеет > 3 → форма невалидна

**Визуал пула:**
```
Пол = [◆] [◆] [◆] [◆] [◆]   ← все 5 свободны
                              ↓ Teacher нажимает + у Анвара
Пул = [◆] [◆] [◆] [◆] [○]   ← 1 израсходован
                              animate-pool-deplete на ◆→○ переходе
```

---

### 8.6 Late Request — countdown timer

Когда запрос одобрен и `unlockedUntil` установлен:
```
✅ Доступ открыт ещё  [18:23:45]  (countdown)
```
- Обновляется каждую секунду на клиенте
- При истечении: страница автоматически переходит в read-only state
- Toast: "Время доступа истекло"

---

### 8.7 Teacher: "мои данные" фильтрация

Teacher никогда не видит данные других учителей:
- Расписание: API возвращает только его уроки (backend filter)
- Студенты: только студенты его групп
- Домашние задания: только его группы
- Поздние запросы: только его запросы

Frontend не должен делать роль-based фильтрацию на клиенте для этих данных —
backend авторизует по JWT и возвращает только разрешённые данные.

UI-side role checks только для:
- Скрытие/показ кнопок (+ Создать, Редактировать, Архивировать)
- Скрытие nav items (Задачи МУП, Аналитика, Компенсации, Настройки)
- Тексты и лейблы ("Мои уроки" для Teacher vs "Все уроки" для MUP)
